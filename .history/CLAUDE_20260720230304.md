# San Pablo City AI Citizen Assistant - System Instructions

## Project Context
You are an expert Frontend AI Engineer building a citizen services web app for San Pablo City (MySanPablo) — a digital gateway for city announcements, emergency hotlines, office directory, and issue reporting. The app must be fast, highly accessible (WCAG 2.1 AA), and resilient under poor mobile data conditions.

## Tech Stack
- **Framework:** React.js + Vite
- **Language:** JavaScript (JSX)
- **Forms & Validation:** React Hook Form + Zod (never use uncontrolled browser inputs or plain useState-based manual validation)
- **UI & Styling:** Tailwind CSS, Lucide React (for icons)
- **Testing:** Vitest + React Testing Library

## Core Coding Principles & Rules

### 1. Robust Form Validation (Strict Rule)
- All user inputs and citizen reports MUST be strictly validated client-side using React Hook Form paired with a declarative Zod schema.
- Show specific, human-readable error messages immediately under the failing input field.

### 2. Accessibility & Mobile-First (WCAG 2.1 AA)
- Every interactive element (inputs, buttons, select menus) must have an explicit and descriptive `aria-label` or `aria-describedby`.
- All custom components must support full keyboard navigation (correct tab-indexing).
- Color contrast ratios must meet at least WCAG AA standards.

### 3. Clean Code & Architecture
- Use plain JavaScript (JSX).
- Prefer functional components with descriptive hook names.
- Organize components under `/src/components`, keep files single-purpose, and colocate a component's test file next to it (e.g. `ReportIssueForm.jsx` + `ReportIssueForm.test.jsx`).
