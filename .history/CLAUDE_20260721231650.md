## Project Rules (React Forms)

1. All forms must use React Hook Form for state management, paired with a Zod 
   schema for validation. Do not use plain useState with manual validation logic 
   — it produces more code and has no guaranteed way to catch every invalid case 
   before submit.

2. Every form input must have accessibility markup by default: a linked <label>, 
   and aria-describedby connecting the input to its error message when invalid. 
   Do not assume this will be added automatically — it must be explicitly built 
   into every form component from the start.

3. No form or feature is considered complete until it has a passing test verifying 
   its core behavior (required fields, validation rules, and success/error states). 
   Code that compiles and looks correct is not proof that it works.
